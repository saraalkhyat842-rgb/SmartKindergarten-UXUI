# Kidoo -UXUI
Graduation project for Digital Egypt Pioneers – UI/UX design of kindergarten tracking app

Project Name: Kidoo

Project Idea:
Mobile application that helps parents monitor their children's progress and daily development in kindergarten.
Through monthly reports, daily activity photos, and teacher notes, parents can stay updated about their child's growth.
The app aims to create an easy communication bridge between parents and teachers using a simple and intuitive interface suitable for everyone.

Team Members:

Sara Abdel Moneim (Team Leader)

Mohamed Magdy

Work Plan:

Research & Analysis

Defined target audience (parents and nursery teachers).

Identified major communication challenges between nurseries and parents.

Built user journeys to understand daily interactions and key pain points.

Information Architecture & Wireframes

Created clear user flows for main app features (Login, Reports, Notifications, Child Profile).

Designed complete low-fidelity and mid-fidelity wireframes.

Visual Identity & Design System

Defined brand colors, typography, and UI components.

Built a full design system ensuring consistency and accessibility across screens.

UI Design

Designed 42 high-fidelity screens in total.

Sara Abdel Moneim → 27 screens.

Mohamed Magdy → 15 screens.

Both collaborated on prototypes, animations, and screen interactions.

Prototype & Testing

Developed the final interactive prototype in Figma using Smart Animate.

Conducted internal testing to ensure usability and visual consistency.

Documentation & Final Presentation

Prepared all project documentation, presentation slides, and supporting visuals jointly.

Roles & Responsibilities:

Sara Abdel Moneim: Project leader, design system, visual design direction, key screens, prototype supervision, presentation.

Mohamed Magdy: Wireframes, UX flow structure, high-fidelity screens, animations, and prototype building.

Both members share responsibility for usability refinement, documentation, and final delivery.

KPIs (Key Performance Indicators)
Metrics to evaluate project success:

Usability Testing Success Rate: Percentage of users completing tasks without external help.

Task Completion Time: Average time taken by users to complete main actions (e.g., checking report, viewing activity).

Error Rate: Frequency of user errors during interaction with the prototype.

User Satisfaction: Post-test rating (scale 1–5).

Adoption Intention: Percentage of users likely to use the app if fully developed.
